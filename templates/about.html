About Vermicelli

Vermicelli is a version control automation system designed to help you seamlessly track and manage the versions of your applications. Built around the widely recognized Semantic Versioning (SemVer) principles, Vermicelli simplifies how you handle your MAJOR, MINOR, and PATCH increments—so you can be confident that every release is clearly documented and easily retrievable.

Why Semantic Versioning?

Semantic Versioning (SemVer) establishes a straightforward convention for assigning version numbers: 

MAJOR.MINOR.PATCH

1. MAJOR: Increment when you introduce changes that are incompatible with previous versions.
2. MINOR: Increment when you add backward-compatible functionality.
3. PATCH: Increment when you fix bugs in a backward-compatible manner.

By adhering to SemVer standards, Vermicelli offers clarity and consistency for both developers and end-users, ensuring everyone knows what to expect from each release.

Key Features

  • Intuitive Version Management: Control version numbers directly through a user-friendly web interface or via API requests, giving you the flexibility to integrate versioning into your existing CI/CD pipeline.
  • Seamless Microservices Integration: Easily track multiple microservices across different repositories or teams. Vermicelli automatically keeps them in sync with minimal effort.
  • Clear Release History: Every time a new version is released, Vermicelli logs the changes and timestamps, providing a transparent audit trail for your entire team.
  • Stable and Predictable Releases: By following SemVer standards, your users and developers can anticipate the impact of each release, reducing risks and improving overall software quality.

Who Can Benefit?

  • Microservice Architecture: If you manage numerous microservices with frequent updates, Vermicelli’s automation and centralized versioning simplify maintaining clarity across all services.
  • Agile Teams: Working in rapid sprints? Vermicelli’s straightforward versioning cuts down on confusion, ensuring features and patches are properly documented as your codebase evolves.
  • Open Source Projects: Transparency and predictability are critical in open source. Vermicelli’s adherence to SemVer helps contributors and community users understand the scope of every change.

Get Started

  1. Configure Your Repo: Integrate Vermicelli with your repository to start automating your version management.
  2. Set Up Webhooks/CI: Tie Vermicelli into your CI/CD pipeline or use its simple API to trigger version increments automatically.
  3. Deploy with Confidence: Release new versions knowing you have a clear history and a predictable versioning strategy.
